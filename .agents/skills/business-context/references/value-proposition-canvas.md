# Value Proposition Canvas Reference

## Purpose

Framework for designing value propositions that match customer needs. Use during Discovery to ensure Solutions address real user jobs, pains, and gains — directly connecting to Value and Usability risks.

## Customer Profile

### Customer Jobs
- **Functional Jobs**: Tasks customers are trying to get done
- **Social Jobs**: How customers want to be perceived by others
- **Emotional Jobs**: How customers want to feel
- **Supporting Jobs**: Ancillary tasks in the context of the main job

### Pains
- **Undesired outcomes**: What frustrates customers about current solutions?
- **Obstacles**: What prevents customers from getting jobs done?
- **Risks**: What could go wrong?
- **Severity**: Rate each pain from moderate to extreme

### Gains
- **Required gains**: Minimum expectations that any solution must meet
- **Expected gains**: What customers normally expect
- **Desired gains**: What would delight customers beyond expectations
- **Unexpected gains**: What would exceed all expectations

## Value Map

### Products & Services
- List of products/services that create value
- What you offer to help customers get jobs done

### Pain Relievers
- How your products/services eliminate or reduce specific pains
- Map each pain reliever to a specific customer pain
- Rank by importance to customer

### Gain Creators
- How your products/services create customer gains
- Map each gain creator to a specific customer gain
- Rank by relevance to customer

## Fit Assessment

The goal is **Problem-Solution Fit**: your Value Map addresses the most important elements of the Customer Profile.

- [ ] Customer segment clearly identified
- [ ] Customer jobs well understood and prioritized
- [ ] Top pains identified and validated with evidence
- [ ] Top gains identified and validated with evidence
- [ ] Pain relievers address the most critical pains
- [ ] Gain creators deliver the most valued gains
- [ ] Differentiated from competitor value propositions

## Value Proposition Statement

**For** [target customer]
**Who** [statement of need or opportunity]
**Our** [product/service name]
**Is a** [product category]
**That** [key benefit, reason to buy]
**Unlike** [primary competitive alternative]
**We** [statement of primary differentiation]

## Application in nautilus

### Connection to OST
- **Customer Jobs** → surface **Opportunities** (unmet jobs = opportunities)
- **Pains** → generate **Hypotheses** about what causes friction
- **Gains** → inform **Solution** design (what would delight users)
- **Pain Relievers** → validate **Value risk** (do we actually relieve the pain?)
- **Gain Creators** → validate **Usability risk** (is the experience good enough to create gains?)

### Common Patterns

**Convenience/Time-Saving**: Process is slow → save time, simplify workflow
**Cost Reduction**: Current solution is expensive → reduce costs, improve ROI
**Performance/Quality**: Current solutions underperform → better results
**Risk Mitigation**: High risk of failure → reduce risk, increase reliability
**Accessibility**: Solution only available to few → make available to many
